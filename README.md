# LegoClassification
過程 (Process):
資料載入與預處理：訓練前，已將圖片資料（train_Data 和 validation_Data）預處理成 NumPy 陣列，並將標籤（trainLabel 和 validationLabel）準備好。
模型編譯：在訓練開始前，模型已經用 Adam 優化器、sparse_categorical_crossentropy 損失函數和 accuracy 指標進行了編譯。
回調函數設定：ModelCheckpoint 回調函數被設定為監控驗證準確度 (val_accuracy)，並在每次驗證準確度提高時保存最佳模型到 model.h5 檔案。
訓練迴圈：model.fit() 函數開始執行訓練。模型將迭代 10 個 epochs (訓練週期)。
批次訓練：在每個 epoch 中，訓練資料會被分成大小為 4 的小批次 (batch_size = 4)。每個批次會被送入模型進行前向傳播，計算預測值和損失。
反向傳播與優化：根據計算出的損失，模型會執行反向傳播來計算梯度，然後 Adam 優化器會使用這些梯度來更新模型的權重，以減少損失並提高準確度。
驗證：在每個 epoch 結束時，模型會使用獨立的驗證資料集 (validation_Data, validationLabel) 來評估其性能，計算驗證損失和驗證準確度，並將結果輸出。
最佳模型儲存：ModelCheckpoint 會檢查當前 epoch 的驗證準確度是否超越歷史最佳值，如果是，則保存當前模型。
重複：此過程會重複 10 次，直到所有 epoch 完成。

方法 (Method): 這段程式碼使用 model.fit() 方法來執行模型的訓練。這是一個高階 API，它封裝了深度學習模型訓練的標準流程，包括：
迭代訓練：在多個 epoch 中重複處理資料。
批次處理：將資料分成小批次進行訓練，以優化記憶體使用並穩定訓練過程。
損失計算與優化：自動計算損失並使用指定的優化器更新模型權重。
回調函數：支持在訓練過程中插入自定義邏輯，例如模型保存、學習率調整等。
實時監控：提供訓練和驗證損失以及準確度的實時輸出。

目的 (Purpose): 這段程式碼的主要目的是訓練一個基於 DenseNet121 的圖像分類模型，使其能夠識別不同種類的樂高迷你公仔。通過在訓練資料上學習，並在驗證資料上評估，模型的目標是實現：
學習複雜特徵：從樂高迷你公仔圖片中提取並學習有效的視覺特徵。
精確分類：建立一個能夠將輸入圖片準確分類到正確公仔類別的模型。
泛化能力：確保模型不僅在訓練資料上表現良好，也能在未曾見過的樂高圖片上做出準確預測，避免過度擬合。

結果 (Result): 從執行輸出可以看到，模型經過 10 個 epoch 的訓練，性能逐步提升：
訓練損失 (Train Loss)：從 4.2432 顯著下降到 0.3271，這表明模型在學習訓練資料的過程中越來越「自信」且錯誤率越來越低。
訓練準確度 (Train Accuracy)：從 0.0570 大幅提升到 0.9689，說明模型對訓練資料的識別能力極強，幾乎可以完美地分類訓練集中的圖片。
驗證損失 (Validation Loss)：從 3.7306 降低到 1.0207，表示模型在未見過的資料上的預測誤差也逐漸減小。
驗證準確度 (Validation Accuracy)：從 0.1067 穩定提升到 0.7640，這是一個重要的指標，它說明模型具有一定的泛化能力，能夠在新的樂高迷你公仔圖片上達到約 76.4% 的識別準確度。ModelCheckpoint 也成功捕捉到每個最佳表現的模型。
